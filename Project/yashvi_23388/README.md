# Multi-Robot Exploration using Entropy-Based Frontier Selection

## Description

This project implements a multi-robot exploration system where multiple robots explore an unknown environment using entropy-based frontier selection. Each robot maintains its own local occupancy grid map, senses its surroundings, communicates with nearby robots, and dynamically assigns goals to maximize information gain.

A key improvement over the baseline implementation is the incorporation of a reachability-aware penalty during goal selection, which prevents robots from selecting infeasible goals and improves overall exploration efficiency.

## Implementation Details

- Number of robots: 4  
- Map size: 100 × 100  
- Sensing radius: 5  
- Communication range: 10  
- Maximum steps: 730  

Robots explore the environment using frontier detection and entropy-based goal selection. They communicate when within range to share map information and improve exploration efficiency.

## ROS2 / RViz Simulation

The project also includes a ROS2-based simulation for visualization in RViz. The file is inside src/my_py_pkg/my_py_pkg

### Steps to Run

#### 1. Open VMBox in lab PC. Open Terminal and Navigate to Workspace

cd ~/ros2_ws

#### 2. Source the Workspace

source install/setup.bash

#### 3. Run the ROS Node

ros2 run my_py_pkg map_node

You should see output such as:

Robot 0 goal: ...
Map published

#### 4. Open RViz (in a new terminal)

cd ~/ros2_ws
source install/setup.bash
rviz2

---

### RViz Configuration

In RViz:

- Set Fixed Frame to `map`

Add the following displays:

- Map  
  Topic: /map  

- Marker  
  Topic: /robot_markers  

---

### Expected Output

- Occupancy grid map updating in real time  
- Colored circles representing robots  
- Colored cubes representing robot goals

### Troubleshooting

ros2 topic list

Ensure the following topics are available:

/map  
/robot_markers
